# GitHub Pages Configuration

This `docs/` folder contains the static HTML dashboard for hosting on GitHub Pages.

## Setup Instructions

### Step 1: Enable GitHub Pages in Repository Settings

1. Go to your repository: `https://github.com/akv803101/Brahma`
2. Click **Settings** → scroll to **Pages** section
3. Under "Build and deployment":
   - **Source**: Select `Deploy from a branch`
   - **Branch**: Select `main`
   - **Folder**: Select `/docs`
4. Click **Save**

### Step 2: GitHub will generate your site URL

Within 1-2 minutes, GitHub will publish your site at:
```
https://akv803101.github.io/Brahma/
```

### Step 3: Verify the Site

- Visit the URL above
- Check that all charts and metrics load correctly
- Share the link with stakeholders

## Auto-Updates

The dashboard pulls data from:
- `outputs/data/leaderboard.csv` — Model performance table
- `outputs/data/drift_config.json` — Dataset metadata
- `outputs/charts/` — All visualization PNGs

**After running new pipeline stages**, simply commit and push to update the dashboard:

```bash
git add docs/
git commit -m "Update dashboard with latest results"
git push origin main
```

GitHub Pages will auto-refresh within seconds.

## Troubleshooting

- **Charts not showing?** Ensure paths in `index.html` match your folder structure
- **Site not deploying?** Check GitHub Actions under the "Actions" tab for errors
- **Slow loading?** Charts are embedded directly; large PNGs may take time
